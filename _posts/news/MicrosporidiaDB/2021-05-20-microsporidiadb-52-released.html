New data in {{project}} 52

EBI Pipeline

{{project}} 52 represents the first release that employs the EBI pipeline for analyzing genomes and RNA-Seq data. ({{project}} Analysis Methods) All genomes were run through the EBI pipeline, updating all analyses such as InterproScan and repeat identification. While much of the EBI pipeline is the same as our previous in-house pipeline, these differences are noted:
  • Expression values calculated from RNA-Seq data are now presented as TPM (Transcripts Per Kilobase Million) instead of FPKM (Fragments Per Kilobase Million) or RPKM (Reads Per Kilobase Million). Learn more
  • Signal peptide predictions are now made with SignalP-4.1, updated from SignalP-3.0.
  • All pseudogenes now include automatically predicted UTRs.

New features in {{project}} 52

  • The gene pages of pseudogenes are modified to improved clarity: (helpful image)
    • The Transcripts table in the Gene Models section now indicates that no protein products are produced.
    • The exons in JBrowse views of pseudogenes are colored gray. Blue represents genes on the forward strand and red represents genes on the reverse strand.
    • The Orthology and Synteny section indicates that orthology is not determined for pseudogenes.
    • Protein sequences are no longer available from the Sequences section.
  • Reference organisms are indicated in the Organism parameter on search pages. VEuPathDB chooses a representative genome from each species and aligns all functional genomics data to the reference. In doing so, all data can be viewed and compared in the genome browser against the reference. In some cases, data are aligned to the source organism in addition to the reference. (helpful image)
  • The algorithm for predicting EC Numbers of genes has been improved. Previously, we predicted EC numbers by using the most prevalent EC number in an ortholog group (defined by OrthoMCL) and transferring that number to all other members in the group. In the new algorithm, we use orthology as well as protein sequence/domain comparison to infer EC number assignments. Additionally, we provide multiple forms of evidence that are used to inform each prediction, allowing users to determine their confidence in the prediction. Specifically, the algorithm evaluates three features (length, domain architecture and BLAST similarity to other proteins) of each protein in an OrthoMCL ortholog group. Those features are compared to the features of proteins already assigned an EC number, in order to infer which proteins should be assigned a specific EC number. We also show the number of proteins and the number of genera that are assigned this EC number within the ortholog group, so that the user can assess the prevalence of a given EC number. This new algorithm performs extremely well in predicting previously assigned EC numbers, demonstrating that the algorithm is accurate.
  • Community annotations made in {{project}} via Apollo are now represented on gene pages in the Gene Models graphic. In addition, links from the gene page to Apollo are offered to facilitate community annotations. Example image. Please see our Apollo webinar for instruction.
