We are pleased to announce the release of {{project}} 49.

The URL https://{{project}}.org directs you to the newest version of our interface. The site contains previous VectorBase.org data plus some new tools and functions in an updated and streamlined interface that empasizes easy access to help information. Navigate to https://legacy.{{project}}.org for the previous version of the interface. The legacy interface will remain active for at least one more release.

New features in {{project}} 49

  • A new search is available that returns genes located in non-nuclear genomic sequences. In VectorBase this query is available for the mitochondrial chromosome of six organims Identify Genes based on Genomic Location (Non-nuclear)
  • When revisiting a search page, your previous parameter values are automatically loaded. This is especially useful for the BLAST search which now retains the input sequence.
  • The cytoscape drawing on metabolic pathway (example pathway) on the record pages (from example gene: go to section "14. Pathways and interactions") now contains:
    • a new search function for finding nodes within the drawing. After running a search, the drawing centers on the nodes which match the search term and highlights them with a bold green border.
    • a "Reset Layout" button that will clear all node paintings as well as reset the node positions and zoom levels.
    • a checkbox for toggling your mouse function between zooming the drawing or scrolling the page.
  • Data set and Organism record pages now contain direct links to our Downloads section where the option is available to download genome-scale sequence and annotation files.
  • For this release, we have combined two pages in a single one to provide a summary of all the available genomes. A very similar table is also available from this search strategy. These tables are referred as "Genome Information and Statistics"

Content migration from legacy to beta VectorBase

As we continue to migrate all the content from the legacy site, before it is shut down, we have migrated for this release four ontologies (IDOMAL, MIRO, TADS and TGMA) and a Sand Fly rearing protocol (from the Sand Fly Fellas); all links are now avaialbe from the Related Sites page. A few new links have also been added to the Related Sites list.

New data in {{project}} 49

Omics datasets

  • In VectorBase we have a total of 327 RNAseq datasets. For this release, 52 RNAseq datasets were available in the search strategy system, now we have a total of 96 RNAseq datsets; more datasets will be included in following releases.
  • Our new RNAseq pipeline had a mistake related to the use of the TPM measure, that has now been fixed. We "strongly encourage" all our users to re-run all search queries involving RNAseq datasets, a change in the obtained list of genes and transcripts is expected, we apologize for any inconvinience this might had couse to your research.
  • The organisms with new gene sets (which might involve intron exon boundaries modifications, addition or deletion of exons or complete genes) for this release are 37 (from our total of 44 genomes). This type of change can also impact your search query results. This is normal and expected, as explained below in our Note. The "Genome Information and Statistics" tables (mentioned above in "New Features"), show you which are the 37 updated genomes, e.g. Anopheles gambiae PEST and Aedes aegypti LVP_AGWG are now on versions AgamP4.13 and AaegL5.3, respectively.
  • A new data track called "tRNAscan" is now available in the Genome Borwser. Example position in A. gambiae PEST: AgamP4_3L:28128595..30253766 (2.13 Mb).

Note: Our bimonthly database releases incorporate new data and correct errors in old data when necessary. Changes in annotation and new experimental data may slightly alter your search results by increasing or decreasing the number of hits. When search parameters change with a new release, we invalidate (Ø) the search and ask you to rerun it. When IDs are updated or removed, we map the old IDs to the new ones, remove the old IDs from your Basket, and leave your Favorites page alone.
